# 学生課送り

> 遊び方：URLを開く → （スマホなら）ホーム画面に追加 → 部屋を作ってコードを共有

身内2〜4人で遊ぶデジタルカードゲーム。

---

## ローカル起動

```bash
cd gakusei-okuri
pip install -r requirements.txt
make run
# → http://localhost:8000
```

テスト：
```bash
make test
```

---

## Render へのデプロイ

1. このリポジトリを GitHub に push する
2. [Render](https://render.com) でアカウント作成 → **New Web Service**
3. リポジトリを接続 → `render.yaml` が自動検出される
4. 「Create Web Service」→ デプロイ完了後に表示されるURLを身内に共有

---

## 「遊びたい時にすぐ起動」要件

### キープウォーム（無料スリープ防止）

Render 無料プランは15分アクセスなしでスリープします（復帰30〜60秒）。  
以下の外部cronで `/healthz` を10分ごとにpingすると常時起動を維持できます：

**UptimeRobot（無料）**
1. https://uptimerobot.com で登録
2. **New Monitor** → HTTP(s) → URL: `https://あなたのRenderURL/healthz`
3. Interval: 10 minutes → Save

**cron-job.org（無料）**
1. https://cron-job.org で登録 → New Cronjob
2. URL: `https://あなたのRenderURL/healthz`、間隔: 10分

> 1サービスの場合、月750時間（≒31日24時間）の無料枠に収まります。  
> スリープなしを確実にしたい場合は Render の有料プラン（約$7/月）をご検討ください。

### PWA（スマホのホーム画面に追加）

1. スマホのブラウザでURLを開く
2. iOS Safari: 共有アイコン → 「ホーム画面に追加」  
   Android Chrome: アドレスバーのメニュー → 「ホーム画面に追加」
3. 次回からアイコン1タップで起動

## アーキテクチャ

```
スマホ/PC ── Nginx(Render) ── FastAPI(uvicorn)
                                  │
                            server/rooms.py
                           （インメモリ部屋管理）
                                  │
                           engine/（純粋Python）
                           apply_action() → 新状態
```

**⚠️ インメモリ注意**：サーバー再起動・Renderスリープ復帰でゲームルームが消えます。  
ゲーム中に全員が離席しないようご注意ください。永続化が必要な場合はRooms実装をDBに差し替えてください。
